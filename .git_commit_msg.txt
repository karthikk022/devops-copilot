fix(ci): update-manifests - extract single sha-* tag from multi-line output

BACKEND_IMAGE / FRONTEND_IMAGE outputs from docker/metadata-action@v5
are multi-line (main, latest, sha-abc123, etc.). The original sed
command interpolated the whole multi-line string into a single -e
expression, breaking it at char 87 with "unterminated s command".

Fix: pipe through grep "sha-" | head -1 to pick exactly one tag,
then guard with a skip-if-empty. The sha-* tag is the most
reproducible choice for GitOps.
